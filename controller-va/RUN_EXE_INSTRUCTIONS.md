# 🚀 Запуск SentinelVA Controller EXE

## ✅ Быстрый запуск

### 1. Проверка порта
```bash
# Проверить что порт 600 свободен
netstat -ano | findstr ":600"
```

### 2. Запуск с диагностикой
```bash
# Запустить с проверкой системы
run_with_debug.bat
```

### 3. Обычный запуск
```bash
# Если порт свободен
run_controller.bat
```

## 🔧 Если EXE файл закрывается сразу

### Проблема: Порт 600 уже используется
**Решение:**
1. **Найдите процесс использующий порт 600:**
   ```bash
   netstat -ano | findstr ":600"
   ```
2. **Закройте приложение** или измените ему порт
3. **Или измените порт в settings.ini:**
   ```ini
   [ServerAPI]
   port_server=601
   ```

### Проблема: Отсутствующие зависимости
**Решение:**
```bash
pip install -r requirements.txt
```

### Проблема: Отсутствующие файлы конфигурации
**Проверьте наличие:**
- `settings.ini`
- `devices.json`

## 🛠️ Диагностика проблем

Если EXE файл все еще закрывается, запустите:
```bash
# Полная диагностика
run_with_debug.bat
```

Скрипт покажет:
- ✅ Статус каждого компонента
- ✗ Точное место ошибки
- 📋 Рекомендации по исправлению

## 📋 Структура запуска

```
controller-va/
├── run_server.py              # Основной скрипт запуска
├── run_controller.bat          # Удобный запуск
├── run_with_debug.bat        # Запуск с диагностикой
├── debug_run.py             # Скрипт диагностики
├── diagnose.bat              # Диагностика системы
├── settings.ini               # Конфигурация
├── devices.json              # Устройства
└── app_va/                  # Основное приложение
```

## 🎯 Успешный запуск

При успешном запуске вы увидите:
```
Starting SentinelVA Controller...
Starting Flask server on http://0.0.0.0:600
```

И сервер будет доступен по адресу: **http://localhost:600**

## 🆘 Поддержка

Если проблема не решена:
1. Запустите `run_with_debug.bat`
2. Покажите полный вывод
3. Проверьте системные требования
4. Убедитесь что порт 600 свободен

**Готово к работе!** 🎉
